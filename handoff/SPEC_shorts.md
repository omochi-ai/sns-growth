# SPEC — 自動Shorts 本番化（T-005）

> 前提：D-003＝無料構成優先（VOICEVOX＋ffmpeg）。有料動画AIは使わない（使う必要が出たら要相談）。
> 役割：動画は直接収益でなく**リーチ増幅フィーダー**。プロフィール（linkpage/note）へ送客するのが目的。

## ゴール
投稿テキスト（`content/threads-x-posts.md` / posts.json）から、
顔出しなしの縦型Shortsを**自動生成**し、3媒体（TikTok / Instagram Reels / YouTube Shorts）へ展開できる状態にする。

## 動画仕様
- 解像度：1080x1920（縦）／尺：15〜30秒／無音区間を作らない
- 構成：
  1. フック字幕（冒頭3秒・最重要。例「請求書の催促、AIに任せていい」）
  2. 本文（投稿テキストを2〜4画面に分割し、字幕＋VOICEVOX読み上げ）
  3. CTA（「プロフィールから無料特典」）
- 音声：VOICEVOX（ローカル・無料）。話者は1つ固定（後で選定）
- 背景：単色/グラデ＋大きめ字幕（可読性優先）。後日スクショ素材も可
- BGM：著作権フリー（無料素材）を任意で。なくても可

## 実装要件
- `shorts build <postId>`：1本のmp4を `output/shorts/` に生成（既存PoCを本番化）
- `shorts batch --count N`：未生成の投稿からN本まとめて生成
- 各動画に対応する**キャプション＋ハッシュタグ**を生成し `output/shorts/<id>.txt` に保存
  - ハッシュタグ例：#業務効率化 #AI activ #ひとり社長 #事務 #自動化（媒体ごとに調整）
- VOICEVOX未起動時は明確なエラーメッセージ＋無音フォールバック（PoC同様）

## 投稿フロー（重要・段階設計）
- **第1段階（今）：手動アップロード**。生成したmp4＋キャプションを、ユーザーが各アプリに手動投稿。
  理由：TikTok/Reels/YTの「無料での自動投稿」はAPI制約・規約・コストの問題が大きく、安易な自動投稿はBAN/コスト要因。
- **第2段階（要調査・別タスク）：各プラットフォームの公式API/投稿自動化の可否・規約・費用を調査**し、
  自動投稿は**コスト/規約をDECISIONSで承認後**に実装。無断で有料API化しない。

## コスト注意（D-003 / コスト管理ルール）
- VOICEVOX・ffmpeg・無料素材のみ＝**無料**。
- 有料動画AI・有料BGM・各媒体の投稿自動化API等が必要になった場合は、**実装前にSTATUS.mdで要相談**。

## まず着手すること
1. VOICEVOX起動前提のフル生成（音声付きmp4）を1本完成させ、STATUSへ。
2. キャプション/ハッシュタグ生成を実装。
3. 手動アップロード用に `output/shorts/` を「mp4＋キャプションtxt」のセットで揃える。
